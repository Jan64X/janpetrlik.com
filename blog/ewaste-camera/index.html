Repurposing an E-Waste camera for something else

January 16, 2026
Reverse EngineeringEmbeddedUARTIoTFirmware

Overview

I had this Xiaomi camera (model MJXSJ05CM) for home security, however I kinda was getting annoyed that it did not have local RTSP, so when the sensor died, I didn’t think much of it and thought i’d just put it in the trash bin.

However I kept it in a storage bin in my workshop for a while (I don’t like throwing out technology too much), and when I came across a GitHub repo with sources to build an experimental version of OpenIPC for this camera, I thought to myself “hmmm, linux with root access, what do, caveman brain thinkie”, then when it was like 2am and I couldn’t sleep it hit me like a semi truck “AHA! I’ll host a webserver on this piece of crap!”

The Github repo I found was: NTUT-112-2-self-learning/MJXSJ05CM. So thank you to this random person, because you made my day and by extension are inspiration for this “project”.

So I cloned the repo, built OpenIPC, double checked that the sensor truly was completely cooked, and then I got to work.

Implementation

I’m pretty familiar with embedded devices such as this one, as it’s also Buildroot based, it wasn’t the hardest thing to modify, only adding a minimal defconfig that removes the actual IP Camera stuffs from OpenIPC and adding a cgi-bin script to Busybox’s httpd was required to get it to show me a custom website.

Building and the project

My repository if for some gosh darn reason you want to replicate something a cursed as this: https://github.com/Jan64X/micam-ssc323-websrv. The instructions to build are really easy:

Just clone the repo, run the docker-build.sh bash script, and the script will automatically build a Docker container (with an older version of GCC which is needed for the SigmaStar 4.9 kernel to compile), and use the Docker container to build OpenIPC with my custom webserver stuffs.

However if you don’t already have OpenIPC installed on this camera, you’ll have to install this before you proceed: https://github.com/user-attachments/files/15988739/MJXSJ05CM.zip After you’ve done that and have my custom build ready, just SSH into the camera and flash my two images:

flashcp -v uImage.ssc323 /dev/mtd2
flashcp -v rootfs.squashfs.ssc323 /dev/mtd3

When the camera boots up, it should not have any OpenIPC stuff running and only SSH and the webserver.

Hardware

  • SoC: SigmaStar SSC323 (ARM Cortex-A7 @ 800MHz)
  • RAM: 32MB
  • Storage: 16MB NOR flash
  • Connectivity: WiFi 4 (802.11n), no Ethernet (not good hardware)
